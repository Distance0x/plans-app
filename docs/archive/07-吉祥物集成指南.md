# 吉祥物动画集成指南

## 概述

本项目集成了长崎素世（Nagasaki Soyo）作为吉祥物，支持两种实现方案：

1. **Live2D 模型**（推荐）- 使用 pixi-live2d-display
2. **简化版本**（备用）- 使用 CSS 动画和静态图片

## 快速开始

### 方案一：使用简化版本（无需额外资源）

```tsx
import { MascotWidget } from './components/mascot/MascotWidget';

function App() {
  return (
    <div>
      <MascotWidget size={200} />
    </div>
  );
}
```

### 方案二：使用 Live2D 模型

#### 1. 准备 Live2D 模型

你需要获取长崎素世的 Live2D 模型文件，通常包括：
- `model.json` 或 `model3.json` - 模型配置文件
- `.moc` 或 `.moc3` - 模型数据文件
- 纹理文件（PNG）
- 动作文件（.motion3.json）
- 表情文件（.exp3.json）

**获取途径：**
- 官方资源（如果有授权）
- 社区分享的模型（注意版权）
- 自己制作或委托制作

#### 2. 放置模型文件

将模型文件放置到项目中：

```
src/assets/live2d/
└── soyo/
    ├── model3.json
    ├── soyo.moc3
    ├── textures/
    │   ├── texture_00.png
    │   └── ...
    ├── motions/
    │   ├── idle.motion3.json
    │   ├── focus.motion3.json
    │   └── ...
    └── expressions/
        ├── default.exp3.json
        └── ...
```

#### 3. 使用 Live2D 版本

```tsx
import { MascotWidget } from './components/mascot/MascotWidget';

function App() {
  return (
    <div>
      <MascotWidget
        size={200}
        useLive2D={true}
        modelPath="/assets/live2d/soyo/model3.json"
      />
    </div>
  );
}
```

## 组件 API

### MascotWidget

主要的吉祥物组件，自动根据应用状态切换动画。

**Props:**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| size | number | 200 | 吉祥物尺寸（像素） |
| className | string | '' | 自定义 CSS 类名 |
| useLive2D | boolean | false | 是否使用 Live2D 模型 |
| modelPath | string | - | Live2D 模型路径 |

### SimpleMascot

简化版吉祥物组件，使用 CSS 动画。

**Props:**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| state | MascotState | 'idle' | 吉祥物状态 |
| size | number | 200 | 吉祥物尺寸 |
| className | string | '' | 自定义 CSS 类名 |

### MascotAvatar

Live2D 版本的吉祥物组件。

**Props:**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| state | MascotState | 'idle' | 吉祥物状态 |
| size | number | 200 | 吉祥物尺寸 |
| className | string | '' | 自定义 CSS 类名 |
| modelPath | string | - | Live2D 模型路径 |
| fallbackImage | string | - | 备用静态图片 |

## 状态说明

吉祥物支持 4 种状态：

| 状态 | 说明 | 触发条件 |
|------|------|----------|
| idle | 空闲 | 番茄钟未运行 |
| focus | 专注中 | 番茄钟工作时段 |
| rest | 休息中 | 番茄钟休息时段 |
| celebrate | 完成庆祝 | 任务完成时（持续3秒） |

## 自定义 Hook

### useMascot

管理吉祥物状态的 Hook。

```tsx
import { useMascot } from './hooks/useMascot';

function MyComponent() {
  const { mascotState, triggerCelebration, setManualState } = useMascot();

  return (
    <div>
      <p>当前状态: {mascotState}</p>
      <button onClick={triggerCelebration}>触发庆祝</button>
    </div>
  );
}
```

**返回值:**

- `mascotState`: 当前吉祥物状态
- `triggerCelebration()`: 手动触发庆祝动画
- `setManualState(state)`: 手动设置状态

## 集成到应用

在主布局中添加吉祥物：

```tsx
// src/App.tsx
import { MascotWidget } from './components/mascot/MascotWidget';

function App() {
  return (
    <div className="app">
      {/* 其他内容 */}
      
      {/* 吉祥物 - 固定在右下角 */}
      <div style={{
        position: 'fixed',
        bottom: 20,
        right: 20,
        zIndex: 1000,
      }}>
        <MascotWidget size={150} />
      </div>
    </div>
  );
}
```

## 性能优化

### Live2D 模型优化

1. **模型文件大小**
   - 纹理尺寸建议 ≤ 2048x2048
   - 使用压缩的 PNG 格式
   - 移除不必要的动作和表情

2. **渲染优化**
   - 使用 `backgroundAlpha: 0` 避免不必要的背景渲染
   - 限制帧率（默认 60fps）
   - 在不可见时暂停渲染

### CSS 动画优化

1. 使用 `transform` 和 `opacity` 属性（GPU 加速）
2. 避免频繁的重绘和回流
3. 使用 `will-change` 提示浏览器优化

## 故障排查

### Live2D 模型无法加载

1. 检查模型路径是否正确
2. 检查模型文件格式（支持 Cubism 2.1 和 4.0）
3. 查看浏览器控制台错误信息
4. 确认所有依赖文件都已正确放置

### 动画不流畅

1. 检查模型文件大小
2. 降低渲染分辨率
3. 使用简化版本替代

### 状态切换不正确

1. 检查 timer-store 和 task-store 是否正常工作
2. 查看 useMascot hook 的逻辑
3. 使用 React DevTools 调试状态
