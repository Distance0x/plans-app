## ▎核心战略调整

### 1. 产品定位重新对齐

**原定位**：番茄钟 + 任务管理 + 日历视图
**新定位**：**可信赖的执行系统** + 专注学习增强

**关键洞察**（来自研究报告）：
> "滴答清单最难复制的并不是'番茄钟'或'日历'，而是'这些模块已经被打磨成一个相对统一的数据与交互系统'"

**战略选择**：
- ✅ 走"平衡跨端优先"路线（最接近滴答清单主流路线）
- ✅ 增强学生场景（专注/锁机/统计）作为差异化
- ❌ 不做 AI 自动排程（留给 v2.0）
- ❌ 不做深度协作（留给 v1.5）

## ▎技术架构调整

### 当前架构问题

1. ❌ **没有提醒引擎** - 任务有截止日期但不会提醒
2. ❌ **没有同步机制** - 数据只在本地，无法跨端
3. ❌ **没有重复规则** - 无法创建重复任务
4. ❌ **没有推送通知** - 无法在后台提醒用户

### 新架构设计

```
┌─────────────────────────────────────────────────────────┐
│ 客户端层 (Electron + React)                             │
├─────────────────────────────────────────────────────────┤
│ - 本地 SQLite 数据库（离线优先）                        │
│ - 操作日志（OpLog）记录所有变更                         │
│ - 本地提醒调度器（基于 node-cron）                      │
│ - UI 层：任务列表、日历视图、番茄钟                    │

                          ↕

│ - 增量同步引擎                                          │
```

## ▎关键设计原则（来自研究报告）

### 1. 提醒系统设计

**三层提醒策略**：
- **硬提醒**：截止/约定时刻，必须到
- **建议提醒**：今日计划开始前的温和提醒
- **汇总提醒**：晚间复盘、早晨计划 digest

**研究依据**：
> "减少通知引发的打断有利于提升表现并降低压力，而频繁中断本身会带来更高的时间压力和应激"

**实施建议**：
- 默认只开启硬提醒
- 提供静默时段设置
- 支持批量汇总通知
- 锁屏内容可隐藏

### 2. 计划页交互设计

**支持 Implementation Intentions**：
- 标题：做什么
- 何时：什么时间
- 做多久：预计时长
- 如果/那么：触发条件

**研究依据**：
> "清晰计划、高质量计划提示以及保留缓冲时间，会比只列清单更能提高执行率"

**实施建议**：
- 任务表单包含：标题、时间、时长、重复规则
- 日历视图显示时间块
- 支持拖拽调整时间和时长
- 自动计算缓冲时间

### 3. 专注模式设计

**学生场景增强**：
- 锁屏模式（专注时锁定应用）
- 白名单（允许访问的应用）
- 热力图（专注时长可视化）
- 自习室（可选，v1.5）

**研究依据**：
> "学生更看重专注、锁机、白名单、自习室与低价"
